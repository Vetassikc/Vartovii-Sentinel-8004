Problem

Autonomous trading agents need a control layer before capital moves.

Agents can act faster than humans, but speed without bounded execution scope creates failure modes that are hard to audit after the fact.

Sentinel-8004 focuses on the last safe checkpoint before execution.
Before execution

Evaluate the trade intent before the venue or chain call happens.

Fail closed

Block when required inputs or safe execution proof are missing.

Machine readable

Return structured outputs judges can inspect without private systems.

Thesis

Sentinel-8004 is not another trading bot.

  • Guardrail first: approve, deny, or downsize a proposed trade.
  • Bounded execution: attach a signed permit envelope to the verdict.
  • Auditable proof: surface a validation artifact that binds registration, decision hash, and permit hash.

Positioning

We do not build bots for capital deployment. We build guardrails for bots that move capital.

Product Flow

One narrow path from intent to execution gate.

The public repo stays intentionally narrow: evaluate the trade, emit the proof objects, and verify that the requested execution remains within the approved envelope.

1. Trade Intent

Agent, venue, market, side, size, and required signals.

2. Policy Evaluation

Configured checks return ALLOW, DENY, or ALLOW_WITH_DOWNSIZE.

3. Signed Verdict

Decision hash, permit hash, signer, expiry, and scope payload.

4. Validation Artifact

Registration binding plus proof status: VALIDATED, CONSTRAINED, or BLOCKED.

5. Permit Verification

Execution is allowed only if the request still fits the signed envelope.

Public-safe boundary

  • No exchange execution, private bridge logic, or hidden operator surface.
  • Judges can inspect the exact JSON objects that drive the guardrail story.
  • When the repo cannot prove a safe path, the demo stays fail closed.

Proof Artifacts

Three objects make the control story inspectable.

Signed Verdict

Execution decision

Machine-readable output that records the verdict and the bounded permit envelope.

decision_hash permit_hash signature expires_at

Validation Artifact

ERC-8004-facing proof layer

Public-safe artifact that binds registration scope, verdict scope, and permit scope for the demo.

registration_id proof_status artifact_hash demo_only

Permit Verification

Last execution gate

Deterministic check that the requested execution still stays inside the approved notional and scope.

permit_valid executable verification_code checks[]

Demo-only boundary

The proof artifacts are real public demo objects, but they do not claim live on-chain registry verification or live capital execution.
